# Data Dictionary

This document defines the synthetic datasets generated by supply_chain_analytics.py.

## Data Generation Summary

- Source type: Fully synthetic (simulated).
- Time grain: Weekly.
- Time horizon: 52 weeks.
- Locations: 12 warehouses.
- Products: 15 SKUs.
- Core unit of analysis:
  - Demand file: week x warehouse x product.
  - Inventory file: warehouse x product.
  - Reorder file: filtered warehouse x product where reorder condition is met.

Description: Weekly demand records for each warehouse and SKU combination.

Expected row count:
- 52 weeks x 12 warehouses x 15 products = 9,360 rows.

## Categorical Value Definitions

### status
- Critical: weeks_of_stock <= 1
- Low: 1 < weeks_of_stock <= 2
- Healthy: 2 < weeks_of_stock <= 6
- Excess: weeks_of_stock > 6

### urgency
- URGENT: lead_time_days >= 12
- High: lead_time_days >= 9 and < 12
- Normal: lead_time_days < 9

## Notes For Data Audiences

- This dataset is synthetic and suitable for modelling demonstrations, prototyping, and dashboard development.
- Values are deterministic for a fixed random seed, supporting reproducible analysis.
- For production decisions, replace synthetic demand and inventory with ERP/WMS source data while preserving the same schema for compatibility.
